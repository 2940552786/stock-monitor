## 方案：新增第⑥条件 — VWAP斜率衰减过滤

### 核心思路

当前5个条件都是**绝对值判断**（斜率是否小于某阈值），但真顶和假顶的区别在**斜率的变化方向**：

- **假顶**（单边上涨中途）：VWAP斜率依然强劲，短暂波动后价格继续上冲
- **真顶**（趋势衰竭）：VWAP斜率已经从高位**明显减速**

新增的条件⑥通过比较**近期VWAP斜率** vs **稍早期VWAP斜率**来判断趋势是否在衰竭。

### 具体算法

```
对于顶信号，计算：
  recent_slope  = 近5分钟VWAP涨速 (%/min)
  earlier_slope = 前5分钟VWAP涨速 (%/min)
  decay_ratio   = recent_slope / earlier_slope

如果 earlier_slope > 0.01%/min（之前确实在涨）
  且 decay_ratio >= 0.5（近期涨速仍有之前的50%以上 → 趋势未衰竭）
  → 过滤掉该信号

否则（斜率已大幅衰减 或 之前本就不在涨）
  → 保留信号
```

底信号镜像逻辑。

### 设计原则：宁可少杀不能错杀

1. **高衰减阈值 (0.5)**：只有近期斜率仍有之前的50%以上才过滤，意味着只有"趋势明显未衰竭"时才杀
2. **最小斜率门槛 (0.01%/min)**：如果之前VWAP基本走平，不触发过滤
3. **数据不足时放行**：K线不够时不判断，保留信号
4. **只做减法**：新条件只在现有条件全部满足后额外过滤，不新增信号

### 修改文件

1. **`server.py`** (line ~700-712)：在信号检测循环中增加条件⑥，同时在顶部新增3个可配置参数
2. **`index.html`** (line ~540-548)：前端 `detectTopBottom()` 同步增加相同逻辑

### 参数（可后续调优）

| 参数 | 默认值 | 含义 |
|------|--------|------|
| `EX_WIN` | 5 | 斜率窗口(分钟) |
| `EX_RATIO` | 0.5 | 衰减比阈值，越低越严格 |
| `EX_MIN_SLOPE` | 0.01 | 触发过滤的最小早期斜率(%/min) |

### 预期效果

- 单边上涨中连续出现的假顶信号，前几个会被过滤（VWAP还在快速拉升）
- 接近真顶时VWAP涨速已经明显放缓，decay_ratio会降到0.5以下，信号保留
- 非趋势行情中（VWAP走平），earlier_slope不满足min条件，不影响现有信号
