
# 集合竞价数据显示 + 手机适配 — 实现计划

## 概述
在分时图中增加早盘集合竞价（9:15-9:25）和尾盘集合竞价（14:57-15:00，仅深市）的可视化展示，同时保证手机端显示效果良好。

## 2. 前端改动 (`index.html`)

### 2.1 `updateTrendChart()` 改造

**a. 扩展 x 轴时间范围**
将竞价时间点（"09:15", "09:18", ..., "09:25"）插入到 `times` 数组最前面（深市尾盘同理追加到最后），使图表覆盖竞价时段。

**b. 新增竞价价格线系列**
- **颜色**: 橙色 `#f0883e`，手机端加粗到 2px（桌面 1.5px）提升辨识度
- **线型**: 虚线 `type: [6, 4]`
- **仅在竞价时段有数据**，其余位置为 `null`

**c. 竞价时段区域着色（markArea）**
使用 ECharts `markArea`：
- 早盘区 (09:15-09:25): `rgba(240,136,62,0.08)`
- 尾盘区 (14:57-15:00): `rgba(240,136,62,0.06)`，仅深市

**d. 竞价关键节点标注（markPoint）**
在 9:25（开盘价确定时刻）标注最终撮合价，显示匹配量。

### 2.2 股票信息栏展示竞价数据
在 `#stock-bar` 中新增竞价摘要行：
- **竞价款**: 早盘竞价最终价 + 匹配量（如 "竞 ¥10.50 5万手"）
- 桌面端：放在现有指标行中
- 手机端（≤768px）：紧凑一行显示，font-size 11px

### 2.3 手机适配要点

**a. 图表尺寸与可读性**
- 手机端竞价虚线宽度 2px（比桌面 1.5px 略粗）
- 竞价区域着色透明度适当提高（0.10）以保证小屏可见
- markPoint 标注字号 9px（桌面 10px），避免遮挡价格线

**b. x 轴标签密度**
- 手机端竞价时段（09:15-09:25 共10分钟），x轴标签间隔 `interval` 计算需确保不重叠
- 09:15、09:20、09:25 三个关键时间点始终显示

**c. 触摸交互**
- 竞价区域同样支持 dataZoom 缩放和拖拽
- 竞价价格线的 tooltip 在手机端使用更大的触控热区

**d. 信息栏精简**
- 手机端竞价信息放在 `#sb-open` 同行显示（如 "竞价 10.50"），避免新增多行撑高面板
- 竞价量和涨幅合并为一条紧凑信息

**e. 仅深市显示尾盘竞价**
上海市场（6开头、9开头）没有尾盘集合竞价，前端根据 `market` 字段判断。

### 2.4 CSS 媒体查询新增
在 `@media (max-width: 768px)` 中追加：
```css
/* 竞价相关 - 手机端 */
.auction-info-mobile { font-size: 11px; color: var(--orange); }
```
