## 方案：结合盘口买卖挂单增强顶底预测

### 改动文件
- `server.py`：增强 3 个 API 端点
- `index.html`：增强分时图和信号展示

### 具体改动

#### 1. `server.py` — `/api/quote` 增加盘口字段
- 解析新浪API返回的五档买卖价量（索引10-29）
- 新增返回字段：`bids[]`, `asks[]`, `bid_total`, `ask_total`, `order_imbalance`（委比）
- 每次查询盘口时存入 `orderbook_history` 时序缓存（保留242个快照≈全天）

#### 2. `server.py` — `/api/trend` 返回委比时序
- 从缓存读取该股票的委比历史
- 新增 `orderbook_timeline` 字段，包含 `[{time, imbalance}, ...]`
- 供前端绘制委比副图

#### 3. `server.py` — `/api/signals` 盘口确认逻辑
- 量价检测到顶/底后，结合当前盘口委比判断：
  - 顶 + 委比< -20%（卖压）→ 强顶 (confidence: high)
  - 底 + 委比> +20%（买盘）→ 强底 (confidence: high)
  - 盘口反向 → 过滤不产生信号
  - 盘口中性 → 普通信号

#### 4. `index.html` — 分时图新增委比副图
- ECharts 双 grid 布局：上方75%=价格+均价，下方25%=委比柱状图
- 红柱=负委比（卖压），绿柱=正委比（买盘）
- 0轴参考线，X轴时间与主图对齐联动

#### 5. `index.html` — 信号分级展示
- 强顶/强底 → 深色箭头 + ⚡标记
- 普通顶底 → 保持现有样式
- 信号流中显示委比值和确认状态
