## 多用户账户系统方案

### 数据存储

**`users.json`** — 用户数据库：
```json
{
  "user1": {
    "password": "sha256_hash",
    "webhook": "https://qyapi.weixin.qq.com/...",
    "watchlist": [{"code":"600519","market":"sh","name":"贵州茅台"}, ...],
    "confirmed_signals": {"600519": {"tops":[...], "bottoms":[...]}},
    "pushed_signals": ["600519|09:50", ...]
  }
}
```

### 服务端改动

1. **新增 3 个 API**：`POST /api/auth/register`、`POST /api/auth/login`、`POST /api/watchlist/sync`
2. **改造 `/api/signals`**：根据登录用户的 token 从 `users.json` 读取该用户的 watchlist、confirmed_signals、pushed_signals、webhook，请求完写回
3. **`push_token.txt` 废弃**，每个用户用自己的 webhook

### 前端改动

1. **登录/注册页面**：未登录时显示，登录后隐藏
2. **登出按钮**：header 右上角
3. **自选股存服务端**：addStock/removeStock 调 `POST /api/watchlist/sync`
4. **所有 API 调用加 token 头**

### 文件结构
```
users.json          ← 新增，用户数据库
server.py           ← 新增3个auth路由，改造signals路由
index.html          ← 新增登录UI + 登出按钮
push_token.txt      ← 删除
```

### 注册流程
用户输入：用户名 + 密码 + 企业微信机器人URL → 注册成功 → 自动登录

### 改动量
- server.py：~80 行新增
- index.html：~50 行新增（登录表单）+ ~20 行修改（加 token）
